Agent Events

A simple, open format for giving agents scheduled workflows and event-driven automation.

Agent Events are folders containing event definition that agents can discover and execute automatically.

What

Skills and tools define what agents can do.

When

Events define when those capabilities are applied.

Why Agent Events?

Agents are increasingly capable, but typically operate in request-response mode without the ability to schedule future actions or maintain long-running workflows. Agent Events solve this by giving agents a standardized way to define, discover, and execute time-based and event-driven workflows.

For event authors

Build scheduled workflows once and deploy them across multiple agent products.

For compatible agents

Support for events lets end users give agents autonomous, scheduled capabilities out of the box.

For teams and enterprises

Capture organizational workflows in portable, version-controlled, auditable packages.

What can Agent Events enable?

  • Scheduled workflows: Daily reports, weekly summaries, monthly reviews - all automated on a cron-like schedule.
  • Proactive monitoring: Check system health, scan for issues, send alerts when thresholds are exceeded.
  • Recurring tasks: Team status emails, standup reminders, ticket triage - consistent and on time.
  • Event-driven automation: React to external triggers like CI/CD failures, customer tickets, or calendar events.
  • Interoperability: Reuse the same event across different event-compatible agent products.

Key Concepts

Schedule

Cron-format schedule determines when the event fires. Supports standard cron syntax plus human-readable aliases.

Skills Integration

Events can reference Agent Skills (local or global) to compose complex workflows from reusable components.

Instruction

Natural language instructions tell the agent what to do when the event fires - no code required.

State & Context

Events can maintain state between runs, access execution history, and adapt based on previous outcomes.

Event Types & Trigger Mechanisms

Every event answers a fundamental question: "When should it be worth considering an action?" Time-based scheduling is one answer, but Agent Events supports multiple trigger mechanisms to enable sophisticated automation patterns.

1. Time-Based Events

Trigger: Scheduled time intervals

Deterministic execution based on cron expressions or natural language schedules. Ideal for recurring tasks with predictable timing requirements.

type: scheduled
schedule: "0 9 * * MON-FRI"

Use cases: Daily reports, periodic health checks, scheduled reminders

2. Manual Events

Trigger: Explicit user invocation

Human-initiated execution through chat interface, API calls, or UI interactions. Enables on-demand workflow execution and event testing without waiting for scheduled triggers.

type: manual
description: Execute on explicit user request

Use cases: One-off reports, testing workflows, user-initiated actions

3. Event-Triggered Events

Trigger: Completion of another event

Reactive execution based on internal event bus signals. Enables event chaining and sequential workflow composition without complex orchestration logic.

type: event-triggered
on:
  event: daily-report-complete

Use cases: Post-processing pipelines, notification chains, dependent workflows

4. State-Change Events

Trigger: Observable state threshold crossing

Condition-based execution when monitored metrics or state variables cross defined thresholds. Enables proactive monitoring and anomaly detection.

type: state-change
condition:
  metric: disk_usage_percent
  operator: ">"
  value: 80

Use cases: Resource alerts, threshold monitoring, anomaly detection

5. External Signal Events

Trigger: Webhooks, API callbacks, MCP server signals

Integration with external systems through webhooks, Model Context Protocol (MCP) servers, or third-party API notifications. Enables real-time response to external state changes.

type: external
source: github.webhook
filter:
  event_type: pull_request

Use cases: CI/CD integration, third-party notifications, real-time system updates

6. User-Activity Events

Trigger: User presence, behavior, or interaction patterns

Execution based on user actions, presence detection, or interaction history. Creates responsive, context-aware automation that adapts to user behavior.

type: user-activity
user: specified_user
activity: session_start

Use cases: Personalized greetings, activity-based reminders, user-context automation

7. Absence Events

Trigger: Expected signal or event does not occur

Proactive detection of missing signals or non-occurrence of expected events within specified time windows. Transforms passive monitoring into proactive anomaly detection.

type: absence
watch:
  event: expected-status-update
  within: 1 hour

Use cases: Dead-man switches, SLA monitoring, missing data detection

8. Composite Events

Trigger: Multiple conditions satisfied simultaneously

Complex event correlation combining multiple trigger types with logical operators. Enables sophisticated automation patterns requiring multi-factor condition evaluation.

type: composite
all:
  - event: daily-report-complete
  - condition:
      metric: error_count
      operator: ">"
      value: 0

Use cases: Multi-condition alerts, complex workflow triggers, correlated event processing

Design Principle: Each event type maintains simplicity by answering a single question about timing or conditions. Complexity emerges from composition, not from individual event definitions. This approach preserves agent autonomy while enabling sophisticated automation patterns.
